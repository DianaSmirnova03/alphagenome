
## Описание файлов

### `prepare_data.py`
- Загружает GTF-аннотацию (GENCODE v39) с помощью `pyranges`.
- Строит два словаря:
  - `genes_dict` – координаты всего гена (хромосома, старт, конец, цепь) для добавления в CSV.
  - `gene_exons.pkl` – список экзонов для каждого гена, используется для построения масок.
- Добавляет к исходным CSV (`train_variants.csv`, `val_variants.csv`) колонки с координатами гена и сохраняет новые файлы `train_with_genes.csv`, `val_with_genes.csv`.

### `data_loader.py`
- Класс `VariantDataset`:
  - Загружает CSV с вариантами и координатами генов.
  - Для каждого примера извлекает из FASTA последовательность длиной `WINDOW` (20 кб) с центром в позиции варианта, учитывая цепь (`strand`).
  - Строит бинарную маску экзонов (`gene_mask`) на основе координат экзонов из `gene_exons.pkl`. Маска равна 1 на позициях, перекрывающихся с экзонами целевого гена.
  - Кодирует последовательности в one-hot с помощью `DNAOneHotEncoder` из `alphagenome_research`.
  - Генератор `generate()` возвращает батчи в виде словаря:
    - `ref_seq`, `alt_seq` – one-hot последовательности (batch, WINDOW, 4)
    - `gene_mask` – маска экзонов (batch, WINDOW)
    - `targets` – z-оценки (batch,)

### `promoter_head.py`
- Кастомная голова `PromoterEffectHead`, наследующая `CustomHead` из `alphagenome_ft`.
- В `predict`:
  - Получает эмбеддинги с разрешением 1 п.н. (`embeddings.get_sequence_embeddings(resolution=1)`).
  - Если передана `gene_mask`, усредняет эмбеддинги только по маскированным позициям (среднее по маске). Иначе – глобальное среднее по всему окну.
  - Линейный слой `hk.Linear(1)` выдаёт скалярное предсказание для одной последовательности.
- `loss` возвращает MSE (используется только при создании loss-функции через библиотеку; в нашем коде loss вычисляется вручную в `train.py`).

### `register_head.py`
- Регистрирует кастомную голову в системе `alphagenome_ft` под именем `'promoter_effect_head'` с конфигурацией:
  - `num_tracks=1` (одно скалярное значение)
  - `output_type=OutputType.RNA_SEQ` (подходящий тип для числовой регрессии)
  - пустые метаданные

### `create_model.py`
- Функция `get_model`:
  - Создаёт модель AlphaGenome с указанной версией (по умолчанию `'all_folds'`) и только с нашей головой.
  - Замораживает все параметры, кроме головы, с помощью `model.freeze_except_head(head_name)`.
  - Возвращает объект модели.

### `train.py`
- `create_train_step`: возвращает JIT-компилированную функцию `train_step`, которая:
  - Принимает батч (словарь с `ref_seq`, `alt_seq`, `gene_mask`, `targets`).
  - Выполняет прямой проход для референсной и альтернативной последовательностей с маской.
  - Вычисляет разность предсказаний `diff = pred_alt - pred_ref` и MSE между этой разностью и `targets`.
  - Обновляет параметры через оптимизатор.
- `train_epoch`: итерирует по генератору датасета, вызывает `train_step`, накапливает loss.
- `validate`: аналогично, но без обновления параметров; возвращает средний loss и корреляцию Пирсона между предсказанной разностью и истинными значениями.

### `run_training.py`
- Основной скрипт обучения:
  - Задаёт гиперпараметры (batch size, learning rate, число эпох и т.д.).
  - Создаёт датасеты `train_dataset` и `val_dataset`.
  - Создаёт модель и оптимизатор (`optax.adamw` с клиппингом градиентов).
  - Выполняет цикл по эпохам, логирует loss и корреляцию в TensorBoard и CSV.
  - Сохраняет чекпоинты после каждой эпохи.

### Тесты
Все тесты используют `device = jax.devices('cpu')[0]` для стабильности и избежания проблем с памятью GPU.

- `test_data.py`: проверяет, что даталоадер выдаёт батчи с правильными формами, без NaN, one-hot корректен, маски не пусты.
- `test_model_creation.py`: создаёт модель и проверяет доступность параметров головы.
- `test_forward.py`: выполняет прямой проход на одном батче, проверяет форму предсказаний и отсутствие NaN.
- `test_training_step.py`: выполняет один шаг обучения, проверяет, что loss не NaN и параметры обновляются.
- `test_overfit_small.py`: обучает модель на первых 10–20 примерах с ранней остановкой, строит график изменения loss, проверяет изменение весов. Позволяет убедиться, что модель способна переобучаться.

```bash
# Рекомендуется использовать чистое окружение Python 3.11
conda create -n ag python=3.11
conda activate ag

pip install --upgrade "jax[cuda12]"  # или "jax[cpu]" если нет GPU
pip install git+https://github.com/google-deepmind/alphagenome.git
pip install git+https://github.com/google-deepmind/alphagenome_research.git
pip install alphagenome-ft pandas pyfaidx tqdm tensorboardX pyranges matplotlib
